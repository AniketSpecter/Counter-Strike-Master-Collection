# Security and Responsible Reporting

Do not post passwords, Google or MEGA credentials, private file links, access tokens, personal Windows paths, or other sensitive information in public Issues or Discussions.

For an ordinary launcher bug, use the structured bug-report form and remove private information from logs and screenshots.

For a security vulnerability that could put users at risk, use GitHub's private vulnerability-reporting option under the repository **Security** tab. Include the affected version, impact, reproduction steps, and a safe proof of concept. Please do not publish exploit details before the issue has been reviewed.

Only download installers from this repository's official Releases page and compare the SHA-256 digest before running them.
